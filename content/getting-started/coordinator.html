Getting Started

Getting started as a coordinator

Welcome to TicketVise! Thanks for choosing TicketVise to assist you in managing your course. Here, we will briefly walk through some features to get you started. To see what Students and Agents can do, please visit their getting started pages. If you find any obstacles, or want to give some feedback, feel free to contact us at info@ticketvise.com.

Installing TicketVise

Integrating TicketVise into your course is easy. There are two ways students can access TicketVise. Using a menu item in the course navigation of with a module.

1. Course Navigation

To install Ticketvise into your course navigation head over to the course Settings and navigate to the Navigation tab. At the bottom of the list you should be able to find TicketVise. By dragging TicketVise into the course navigation, you have made TicketVise accesible to your students and teaching assistants.

2. Module

Modules can give more strucutre to your course. To add TicketVise as a module head of to your course Modules. Inside a module create a new item by pressing the add button. Select External tool in the dropdown list. A new list will apear, search for TicketVise and press Add item.

One last step is needed to complete the installation. As a coordinator you have to be the first person to open TicketVise to create your course in the tool!

Congratulations you have now installed TicketVise in Canvas!

Creating an inbox

Creating an inbox to match your course is as simple as creating a link on the canvas page. When you visit this link on the canvas page, you create the course in TicketVise. When students and agents click this link, they too will be added to the course, with the permissions inherited from canvas.

Configuring an inbox

When opening the inbox, you can go to the configuration to change some settings about the course. Most of these settings are self-explaining, we will quickly go over some of the others. Toggling the "Assignee visible to students" changes if the students can see if and who is assigned to their ticket. Replying on a ticket will however always show the author of the reply. You can set a timer for closing a ticket if no reply has been sent for some amount of time, or you can send an alert if no answer has been given after some time. These help to prevent tickets from being unanswered and keep them organised.

Scheduling

To automatically assign tickets to agents, there are two scheduling methods. Currently, there is an option to manually assign tickets, no automatic scheduling takes place. The least assigned first makes sure that every agent is assigned the same amount of tickets, if agent 1 that get a ticket from agent 2 assigned to them will not get another one assigned to them before agent 2 has received a new ticket to compensate for the loss of the first. When using round robin, this will not happen, but all tickets will be assigned to the next TA in the list.

Configuring users

The users tab on the configuration page shows all users connected to the course known by TicketVise. As a coordinator, you can change the permissions of each user in the course and view information about them.

Configuring labels

Labels can be added to the wish of the coordinator. Labels are useful for organizing the tickets inside the course. Changing the visibility of the student or enabling the label in general are ways in which you can control the use of the labels. Labels not visible for students can be used to organise labels within the staff.

Inbox statistics

The statistics from the inbox gives insight in the amount of tickets and the activity of the agents. These insights can help to recognise where the problems lies with the students as well as the agents.
